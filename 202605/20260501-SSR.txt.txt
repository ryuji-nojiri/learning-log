# 学んだこと
■ SSR とは？
ざっくりと。。。「サーバー側で HTML を組み立ててからブラウザに投げる」
〇CSRとの違い
CSR：
　・ブラウザが中身が空っぽの HTML をもらう。
　・ブラウザで JavaScript が動いて、後からコンテンツを作る。
　・最初の表示が真っ白で遅い。
SSR：
　・リクエスト：ユーザーがページにアクセス。
　・サーバー側の処理：サーバーが API を叩いてデータを取得。
　・レンダリング：データを流し込んで HTML を生成。
　・レスポンス：ブラウザに完成した HTML を送る。
　・表示：ブラウザはすぐに表示できる。

〇ハイドレーション（Hydration）
・サーバーから届いた直後の HTML は静的な状態。ボタンなどはまだ動かない。
・後から JavaScript が届き、 HTMl にイベントリスナーなどを結合する。
　→これをハイドレーションという。

〇メリット / デメリット
メリット：
　・最初の表示が爆速：ユーザーがすぐ内容を読める。
　・SEO に強い：クローラーが最初から HTML を読み取れるので、検索順位にいい影響がある。
　・SNS 映え：シェアした時のプレビュー（OGP）がちゃんと出る。
デメリット：
　・サーバ負荷が高い：毎回 HTML を生成するので、アクセスが多いとサーバに負荷がかかる。
　・見えているのに動かない時間がある：HTML が表示されてからハイドレーションが終わるまでの数秒間、ボタンを押しても反応しないことがある。

〇関連用語整理
SSR：リクエストが来るたびにサーバーで作る。最新データの取得に向いている。
SSG：サイト公開前にあらかじめ全部作っておく。爆速だけど更新はビルドが必要。
ISR：SSG と SSR のいいとこ取り。時間経過で古いページを裏側で作り直す。
